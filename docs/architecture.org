* Gowl Architecture
:PROPERTIES:
:CUSTOM_ID: gowl-architecture
:END:
** Overview
:PROPERTIES:
:CUSTOM_ID: overview
:END:
Gowl is a GObject-based Wayland compositor built on wlroots 0.19. Inspired by dwl and dwm, it reimagines their tiling window management philosophy with a full GObject type system, providing a modular, introspectable compositor with a plugin architecture based on GModule and GObject interfaces.

The design philosophy is to expose every compositor subsystem as a well-typed GObject, making it possible for modules to hook into any part of the compositor lifecycle through a set of clearly defined interfaces.

** GObject Type Hierarchy
:PROPERTIES:
:CUSTOM_ID: gobject-type-hierarchy
:END:
The codebase is organized into several layers, each built on GObject fundamentals.

** Module / Plugin System
:PROPERTIES:
:CUSTOM_ID: module--plugin-system
:END:
*** How Modules Work
:PROPERTIES:
:CUSTOM_ID: how-modules-work
:END:
Modules are shared objects (=.so= files) that export a single entry point:

#+begin_src C
G_MODULE_EXPORT GType
gowl_module_register(void)
{
    return MY_TYPE_MODULE;
}
#+end_src

The =GowlModuleManager= loads the =.so= via =g_module_open()=, calls =gowl_module_register()= to get the =GType=, instantiates the module, and classifies it into dispatch arrays based on which interfaces the module implements.

*** Module Lifecycle
:PROPERTIES:
:CUSTOM_ID: module-lifecycle
:END:
1. *Load* -- =g_module_open()= on the =.so= file.
2. *Register* -- =gowl_module_register()= returns the module's GType.
3. *Instantiate* -- =g_object_new()= creates the module instance.
4. *Classify* -- The module manager checks which interfaces the instance implements and adds it to the corresponding dispatch arrays.
5. *Activate* -- =gowl_module_activate()= calls the virtual =activate= method.
6. *Dispatch* -- Events are dispatched to interface-specific arrays in priority order.
7. *Deactivate* -- =gowl_module_deactivate()= on shutdown or removal.

*** Module Priority
:PROPERTIES:
:CUSTOM_ID: module-priority
:END:
Each module has a =priority= property (integer). Dispatch arrays are sorted in ascending priority order (lower value = dispatched first). This allows modules to control ordering, for example a keybind handler module with priority 0 runs before one with priority 100.

*** Module Signals
:PROPERTIES:
:CUSTOM_ID: module-signals
:END:
=GowlModuleManager= emits:

- =module-loaded= -- after successful registration
- =module-activated= -- after successful activation
- =module-deactivated= -- after deactivation
- =module-error= -- on any module operation failure

*** Module Search Paths
:PROPERTIES:
:CUSTOM_ID: module-search-paths
:END:
Modules are loaded from:

1. =$GOWL_MODULEDIR= (compile-time default: =/usr/local/lib/gowl/modules/=)
2. =~/.local/lib/gowl/modules/=
3. The build output directory =build/release/modules/= (for development)

** Interface-Based Hook System
:PROPERTIES:
:CUSTOM_ID: interface-based-hook-system
:END:
Gowl defines 18 GObject interfaces. A module implements one or more of these to hook into compositor events. The =GowlModuleManager= maintains a dispatch array per interface.

*** Interface List
:PROPERTIES:
:CUSTOM_ID: interface-list
:END:
| Interface                 | Methods                                                                                             | Dispatch Semantics                                    |
|---------------------------+-----------------------------------------------------------------------------------------------------+-------------------------------------------------------|
| =GowlLayoutProvider=      | =arrange(monitor, clients, area)=, =get_symbol()=                                                   | First matching provider arranges the monitor.         |
| =GowlKeybindHandler=      | =handle_key(modifiers, keysym, pressed)=                                                            | First handler returning TRUE consumes the event.      |
| =GowlMouseHandler=        | =handle_button(button, state, modifiers)=, =handle_motion(x, y)=, =handle_axis(value, orientation)= | First handler returning TRUE consumes the event.      |
| =GowlClientDecorator=     | =decorate(client)=, =undecorate(client)=                                                            | All decorators are called (broadcast).                |
| =GowlClientPlacer=        | =place(client, monitor)=                                                                            | First placer returning TRUE claims placement.         |
| =GowlFocusPolicy=         | =should_focus(client)=, =focus_changed(client)=                                                     | First policy returning FALSE vetoes focus.            |
| =GowlMonitorConfigurator= | =configure(monitor, mode)=                                                                          | First configurator returning TRUE claims the monitor. |
| =GowlRuleProvider=        | =get_rules()=, =match(client)=                                                                      | All providers are queried; rules are merged.          |
| =GowlStartupHandler=      | =on_startup(compositor)=                                                                            | All handlers called (broadcast).                      |
| =GowlShutdownHandler=     | =on_shutdown(compositor)=                                                                           | All handlers called (broadcast).                      |
| =GowlIpcHandler=          | =handle_command(command, args)=                                                                     | First handler returning a result claims the command.  |
| =GowlTagManager=          | =view_tag(monitor, mask)=, =set_tag(client, mask)=                                                  | First manager returning TRUE claims the operation.    |
| =GowlGapProvider=         | =get_gaps(monitor)=                                                                                 | First provider returning non-NULL gaps wins.          |
| =GowlBarProvider=         | =render_bar(monitor, surface)=, =get_bar_height()=                                                  | First provider returning TRUE claims the bar.         |
| =GowlScratchpadHandler=   | =toggle_scratchpad(name)=, =add_to_scratchpad(client, name)=                                        | First handler returning TRUE claims the operation.    |
| =GowlSwallowHandler=      | =should_swallow(parent, child)=, =swallow(parent, child)=, =unswallow(parent)=                      | First handler returning TRUE claims the swallow.      |
| =GowlStickyHandler=       | =is_sticky(client)=, =set_sticky(client, sticky)=                                                   | First handler returning TRUE claims the operation.    |
| =GowlCursorProvider=      | =get_cursor_image(mode)=, =set_cursor(image)=                                                       | First provider returning non-NULL claims the cursor.  |

*** Dispatch Semantics
:PROPERTIES:
:CUSTOM_ID: dispatch-semantics
:END:
There are two dispatch patterns:

1. *Consumable* -- The event is dispatched to handlers in priority order. The first handler that returns =TRUE= (or a non-NULL result) consumes the event and no further handlers are called. Used by: =GowlKeybindHandler=, =GowlMouseHandler=, =GowlClientPlacer=, =GowlIpcHandler=, etc.

2. *Broadcast* -- Every handler is called regardless of return value. Used by: =GowlStartupHandler=, =GowlShutdownHandler=, =GowlClientDecorator=.

** Configuration System
:PROPERTIES:
:CUSTOM_ID: configuration-system
:END:
*** Dual Config Architecture
:PROPERTIES:
:CUSTOM_ID: dual-config-architecture
:END:
Gowl supports two configuration mechanisms that work together:

1. *YAML Config* (=config.yaml=) -- Declarative configuration for all standard settings, keybinds, rules, and module options. Parsed at startup.

2. *C Config* (=config.c=) -- Compiled to a =.so= and loaded via =g_module_open()=. The exported =gowl_config_init()= function runs after YAML loading, allowing it to override or supplement YAML values programmatically.

*** Config Precedence
:PROPERTIES:
:CUSTOM_ID: config-precedence
:END:
From lowest to highest priority:

1. Built-in defaults (hardcoded in =GowlConfig= property defaults)
2. YAML configuration file
3. C configuration module (=gowl_config_init()=)
4. CLI arguments

*** YAML Search Paths
:PROPERTIES:
:CUSTOM_ID: yaml-search-paths
:END:
The config loader searches for =config.yaml= in this order (first match wins):

1. =./data/config.yaml= (project directory, for development)
2. =~/.config/gowl/config.yaml= (user config, recommended)
3. =/etc/gowl/config.yaml= (system-wide config)
4. =/usr/local/gowl/config.yaml= (installation default)

*** C Config Search Paths
:PROPERTIES:
:CUSTOM_ID: c-config-search-paths
:END:
The config compiler searches for =config.c= in:

1. =~/.config/gowl/config.c= (user config)
2. =/etc/gowl/config.c= (system-wide config)

The compiled =.so= is cached in =$XDG_CACHE_HOME/gowl/config.so=. It is recompiled only if the source is newer than the cached =.so=.

*** Config Properties
:PROPERTIES:
:CUSTOM_ID: config-properties
:END:
All configuration values are exposed as GObject properties on =GowlConfig=, making them accessible via =g_object_get()= / =g_object_set()= and through GObject introspection bindings. The =changed= signal is emitted whenever a property changes, and =reloaded= after a full reload.

** XWayland (X11 Client Support)
:PROPERTIES:
:CUSTOM_ID: xwayland
:END:
When built with =BUILD_XWAYLAND=1= (the default) and the =Xwayland=
server binary is installed, gowl runs its own rootless XWayland so
unmodified X11 applications run inside the session.

- On startup (=gowl_compositor_start=) gowl creates a *lazy*
  =wlr_xwayland= (the X server process only spawns when the first X11
  client connects), binds the seat to it, and exports =DISPLAY= to its
  X display name (e.g. =:1=).  =DISPLAY= is re-asserted after the nested
  libdecor/GTK setup, which can perturb the environment, so children
  reliably inherit gowl's own X server rather than a parent session's.
- X11 windows are represented by the same =GowlClient= type as XDG
  toplevels.  A client is an X11 surface when =xwayland_surface= is
  non-=NULL=; the =client_is_x11()= / =client_surface()= accessors and
  small branches in the shared map / commit / resize / fullscreen /
  focus / destroy paths handle the two surface types uniformly (the
  dwl model).  Managed toplevels are tiled like XDG windows;
  override-redirect surfaces (menus, tooltips) float.
- =gowl_compositor_get_xwayland_display()= returns the X display name so
  integrators (e.g. cmacs's =gowl-spawn=) can point spawned X11 clients
  at it.

Build without it via =make BUILD_XWAYLAND=0= (drops the =xcb= /
=xcb-icccm= dependencies); the X11 code is then compiled out behind
=GOWL_HAVE_XWAYLAND= and the compositor is Wayland-only.

** Systemd user-session bootstrap

=gowl_compositor_start= also bootstraps the systemd user session
(=src/util/gowl-systemd.c=), which =gnome-session= does for GNOME but a
bare compositor launch does not.  It runs
=systemctl --user import-environment= for the live compositor env and
starts the shipped =gowl-session.target= (=Wants==graphical-session.target=,
which itself refuses manual start).  This brings up the
graphical-session-bound user services -- =xdg-desktop-portal=,
=flatpak-session-helper=, =gvfs= -- that flatpak and other graphical apps
depend on (without it, flatpaks get no Settings portal and mis-theme).
=gowl_compositor_quit= stops the target.  Best-effort: no-op where
=systemctl= is absent, and =GOWL_DISABLE_SYSTEMD=1= opts out.

** Raw Frame Sink (External Animated Backgrounds)

Gowl can display *externally produced* frames as the wallpaper or the
lock-screen background without rendering them itself and *without depending on
any rendering engine*.  Only a plain pixel span crosses the boundary, so an
outside producer (e.g. a cmacs build rendering libregnum screensavers --- in
cmacs's case from a *separate render process*, streamed in over shared memory)
keeps all of the GL / engine code on its side.

- =GowlRawBuffer= (=src/core/gowl-frame-sink.{c,h}=) wraps a caller-supplied
  =ARGB8888= buffer =(pixels, width, height, stride)= in a =struct
  wlr_buffer=.  The byte order is =DRM_FORMAT_ARGB8888= — little-endian
  =B,G,R,A=, exactly what =glReadPixels(GL_BGRA)= yields, so no channel
  swizzle is needed.  Pixels are *copied*, so the caller may reuse its buffer
  immediately.
- =GowlFrameSink= owns one =wlr_scene_buffer= per monitor on a chosen scene
  layer.  =gowl_frame_sink_push()= creates-or-swaps the per-monitor buffer
  (cheap =wlr_scene_buffer_set_buffer= on re-push); =clear= / =clear_all= tear
  the nodes down.  A =keep_at_bottom= flag lowers each pushed node to the
  bottom of its layer (used for the lock background so the password UI on the
  same layer stays above it).

The compositor exposes this as:

#+BEGIN_SRC c
void     gowl_compositor_push_wallpaper_frame (self, monitor, px, w, h, stride);
void     gowl_compositor_clear_wallpaper_frame(self, monitor);   /* NULL = all */
void     gowl_compositor_push_lock_frame      (self, monitor, px, w, h, stride);
void     gowl_compositor_clear_lock_frame     (self, monitor);
gboolean gowl_compositor_has_lock_frame       (self);
gboolean gowl_compositor_monitor_bg_covered   (self, monitor);
#+END_SRC

- *Wallpaper frames* go on the =BG= layer, drawn *above* the static wallpaper
  image, so clearing them reveals the original wallpaper.
  =monitor_bg_covered= reports whether a monitor's wallpaper is fully occluded
  by a visible fullscreen client, letting an animated producer pause it.
- *Lock frames* go on the =BLOCK= (session-lock) layer, kept beneath the
  password UI; while a lock frame is present the solid lock backdrop
  (=locked_bg=) is hidden so the frame shows through, and the screenlock
  module renders its password surface transparently (so the box, dots, and
  status text composite on top).  =has_lock_frame= is how screenlock knows.

*Invariant:* none of this links =libregnum=, =graylib=, =raylib=, or GL — the
sink sources only wlroots / glib / pixman.  =tests/test-no-libregnum.sh=
(run by =make test=) fails the build if any =lrg_*= / =grl_*= symbol or engine
header ever leaks into =libgowl= or a module.
