*** wlroots version (0.19 or 0.20)
:PROPERTIES:
:CUSTOM_ID: wlroots-version
:END:
gowl builds against *either* wlroots 0.19 or 0.20.  They ship as
parallel, ABI-incompatible pkg-config modules (=wlroots-0.19= /
=wlroots-0.20=), so a build links exactly one, chosen at build time:

- *Default:* the newest installed wins.  =make= picks =wlroots-0.20= if
  present, otherwise =wlroots-0.19=.
- *Override:* =make WLROOTS=0.19= forces 0.19 (e.g. on Fedora 42/43,
  where 0.20 is not yet packaged), or pins 0.19 on a host that also has
  0.20.

The choice is reported by =make show-config= and surfaced to the code as
=GOWL_WLROOTS_VERSION_{MAJOR,MINOR}= plus the capability macro
=GOWL_HAVE_WLROOTS_0_20= (see =src/gowl-wlroots-compat.h=).

*Why it matters:* per-window *screencast capture* (sharing a single
window via =xdg-desktop-portal-wlr=, e.g. Zoom's "use system window
capture") needs the =ext-image-copy-capture= + =ext-foreign-toplevel=
protocol stack, which exists only in *wlroots 0.20+*.  Built against
0.19, gowl provides monitor (whole-output) capture only; built against
0.20 it provides monitor *and* window capture.  Everything else is
identical between the two.

*** Fedora (dnf)
:PROPERTIES:
:CUSTOM_ID: fedora-dnf
:END:
This is the primary supported platform. If using Fedora Silverblue with a distrobox, run these inside the container.

#+begin_src sh
sudo dnf install -y \
    gcc make pkg-config \
    glib2-devel \
    gobject-introspection-devel \
    json-glib-devel \
    wayland-devel \
    wayland-protocols-devel \
    wlroots-devel \
    libxkbcommon-devel \
    libinput-devel \
    libyaml-devel \
    libxcb-devel \
    xcb-util-wm-devel
#+end_src

On Fedora 44+, =wlroots-devel= is 0.20 (window capture).  Fedora 42/43
ship 0.19 only; install =wlroots0.19-devel= there and build with
=make WLROOTS=0.19= (monitor capture only).  Where both are installed,
=wlroots-devel= (0.20) wins by default.

Or use the built-in Makefile target:

#+begin_src sh
make install-deps
#+end_src

*** Ubuntu 24.04 (apt + wlroots from source)
:PROPERTIES:
:CUSTOM_ID: ubuntu-2404
:END:
Ubuntu 24.04 (noble) packages wlroots 0.17 -- too old for gowl, which
needs 0.19 or 0.20.  Build wlroots from source to a local prefix and
point pkg-config at it.  0.20 is recommended (it unlocks window capture);
substitute =0.19= in the =git checkout= for a monitor-only build.

#+begin_src sh
# Build deps (meson/ninja + wayland/drm/pixman/etc.)
sudo apt install -y \
    build-essential meson ninja-build pkg-config cmake \
    libwayland-dev wayland-protocols libwayland-egl-backend-dev \
    libdrm-dev libgbm-dev libgles2-mesa-dev libegl1-mesa-dev \
    libpixman-1-dev libinput-dev libxkbcommon-dev \
    libudev-dev libseat-dev libdisplay-info-dev hwdata \
    libliftoff-dev libxcb1-dev libxcb-composite0-dev \
    libxcb-icccm4-dev libxcb-res0-dev xwayland \
    glib2.0-dev libgirepository1.0-dev libjson-glib-dev libyaml-dev

# wlroots to /opt/wlroots (keeps it off the system; reversible)
git clone https://gitlab.freedesktop.org/wlroots/wlroots.git
cd wlroots
git checkout 0.20            # or 0.19
meson setup build --prefix=/opt/wlroots -Dexamples=false
ninja -C build
sudo ninja -C build install

# Make pkg-config find it, then build gowl
export PKG_CONFIG_PATH=/opt/wlroots/lib/x86_64-linux-gnu/pkgconfig:$PKG_CONFIG_PATH
export LD_LIBRARY_PATH=/opt/wlroots/lib/x86_64-linux-gnu:$LD_LIBRARY_PATH
pkg-config --modversion wlroots-0.20   # confirm it resolves
cd /path/to/gowl && make                # auto-detects 0.20
#+end_src

Note the multiarch pkgconfig path (=lib/x86_64-linux-gnu/pkgconfig=);
on other arches adjust accordingly.  The same =PKG_CONFIG_PATH= /
=LD_LIBRARY_PATH= must be exported when building cmacs against this
wlroots (its =configure= detects the version the same way).

** Installation
:PROPERTIES:
:CUSTOM_ID: installation
:END:
#+begin_src sh
make install
#+end_src

Installs to =$PREFIX= (default =/usr/local=):

- =$PREFIX/bin/gowl= -- Compositor executable
- =$PREFIX/lib/libgowl.so.*= -- Shared library
- =$PREFIX/lib/libgowl.a= -- Static library
- =$PREFIX/include/gowl/= -- Header files
- =$PREFIX/lib/pkgconfig/gowl.pc= -- pkg-config file
- =$PREFIX/lib/gowl/modules/= -- Module shared objects
- =$PREFIX/share/gir-1.0/= -- GIR file (if =BUILD_GIR=1=)
- =$PREFIX/lib/girepository-1.0/= -- Typelib (if =BUILD_GIR=1=)
- =$PREFIX/lib/systemd/user/gowl-session.target= -- systemd user unit

**** Systemd user-session integration
:PROPERTIES:
:CUSTOM_ID: systemd-session
:END:

A compositor launched straight from the display manager -- gowl, or
=cmacs --gowl= -- does not bootstrap the systemd user session the way
=gnome-session= does.  Without that, =graphical-session.target= is never
pulled in (it has =RefuseManualStart=yes=, so it can only be activated
by a unit that =Wants== it) and the user manager keeps a stale prior
session's environment.  The result: =xdg-desktop-portal= can't satisfy
its =Requisite=graphical-session.target= and stays dead, so flatpak and
other graphical apps get no =org.freedesktop.portal.Settings= and fall
back to a light theme / no file portal / no screencast.

gowl bridges this from =gowl_compositor_start= (=src/util/gowl-systemd.c=):
on startup it runs =systemctl --user import-environment= for the live
=compositor env (=WAYLAND_DISPLAY=, =XDG_CURRENT_DESKTOP=, ...) and starts
the shipped =gowl-session.target= (which =Wants==graphical-session.target=);
on quit it stops the target.  Everything is best-effort -- a host
without =systemctl= is a silent no-op, and =GOWL_DISABLE_SYSTEMD=1= opts
out entirely.

=make install= places =gowl-session.target= in =$PREFIX/lib/systemd/user=,
which the user manager scans for both =$PREFIX=/usr= and =$PREFIX=/usr/local=.
After installing, =systemctl --user daemon-reload= picks it up; the unit is
started automatically the next time gowl starts.

Use =DESTDIR= for staged installs (packaging):

#+begin_src sh
make install DESTDIR=/tmp/gowl-pkg
#+end_src

** Development Workflow
:PROPERTIES:
:CUSTOM_ID: development-workflow
:END:
A typical development cycle:

#+begin_src sh
# Build debug with all modules
make DEBUG=1

# Run tests
make test

# Run the compositor (in a nested Wayland session or TTY)
./build/debug/gowl

# Run with debug logging
./build/debug/gowl --debug

# Run with a custom config
./build/debug/gowl --config ./my-config.yaml
#+end_src
