** Installing Dependencies
:PROPERTIES:
:CUSTOM_ID: installing-dependencies
:END:
*** Fedora (dnf)
:PROPERTIES:
:CUSTOM_ID: fedora-dnf
:END:
This is the primary supported platform. If using Fedora Silverblue with a distrobox, run these inside the container.

#+begin_src sh
sudo dnf install -y \
    gcc make pkg-config \
    glib2-devel \
    gobject-introspection-devel \
    json-glib-devel \
    wayland-devel \
    wayland-protocols-devel \
    wlroots-devel \
    libxkbcommon-devel \
    libinput-devel \
    libyaml-devel \
    libxcb-devel \
    xcb-util-wm-devel
#+end_src

Or use the built-in Makefile target:

#+begin_src sh
make install-deps
#+end_src

*** Quick Build
:PROPERTIES:
:CUSTOM_ID: quick-build
:END:
#+begin_src sh
make

This builds the shared library, static library, executable, and all bundled modules in release mode. Output goes to =build/release/=.

*** Debug Build
:PROPERTIES:
:CUSTOM_ID: debug-build
:END:
#+begin_src sh
make DEBUG=1
#+end_src

Builds with =-g -O0 -DDEBUG=. Output goes to =build/debug/=.

*** Debug Build with AddressSanitizer
:PROPERTIES:
:CUSTOM_ID: debug-build-with-addresssanitizer
:END:
#+begin_src sh
make DEBUG=1 ASAN=1
#+end_src

Adds =-fsanitize=address -fsanitize=undefined= to both compile and link flags.

*** Without XWayland
:PROPERTIES:
:CUSTOM_ID: without-xwayland
:END:
#+begin_src sh
make BUILD_XWAYLAND=0
#+end_src

Disables XWayland support. This removes the =xcb= and =xcb-icccm= dependencies.

*** GObject Introspection
:PROPERTIES:
:CUSTOM_ID: gobject-introspection
:END:
#+begin_src sh
make BUILD_GIR=1
#+end_src

Generates =Gowl-0.1.gir= and =Gowl-0.1.typelib= for use with language bindings (Python, JavaScript, etc.).

*** Running Tests
:PROPERTIES:
:CUSTOM_ID: running-tests
:END:
#+begin_src sh
make test
#+end_src

Builds the test suite (using GLib's GTest framework) and runs all test binaries. Each test binary exercises a specific subsystem:

- =test-boxed= -- Boxed type copy/free, equality, serialization
- =test-config= -- Config property defaults, YAML loading, keybind/rule management
- =test-enums= -- Enum type registration and value validation
- =test-keybind= -- Keybind string parsing and serialization
- =test-layout= -- Layout provider interface dispatch
- =test-module= -- Module lifecycle, registration, activation

** Build Options Summary
:PROPERTIES:

| =DEBUG=          | =0=          | =1= enables debug symbols and =-O0=               |

make DEBUG=1 BUILD_GIR=1 PREFIX=/usr
#+end_src

** Installation
:PROPERTIES:
:CUSTOM_ID: installation
:END:
#+begin_src sh
make install
#+end_src

Installs to =$PREFIX= (default =/usr/local=):

- =$PREFIX/bin/gowl= -- Compositor executable
- =$PREFIX/lib/libgowl.so.*= -- Shared library
- =$PREFIX/lib/libgowl.a= -- Static library
- =$PREFIX/include/gowl/= -- Header files
- =$PREFIX/lib/pkgconfig/gowl.pc= -- pkg-config file
- =$PREFIX/lib/gowl/modules/= -- Module shared objects
- =$PREFIX/share/gir-1.0/= -- GIR file (if =BUILD_GIR=1=)
- =$PREFIX/lib/girepository-1.0/= -- Typelib (if =BUILD_GIR=1=)
- =$PREFIX/lib/systemd/user/gowl-session.target= -- systemd user unit

**** Systemd user-session integration
:PROPERTIES:
:CUSTOM_ID: systemd-session
:END:

A compositor launched straight from the display manager -- gowl, or
=cmacs --gowl= -- does not bootstrap the systemd user session the way
=gnome-session= does.  Without that, =graphical-session.target= is never
pulled in (it has =RefuseManualStart=yes=, so it can only be activated
by a unit that =Wants== it) and the user manager keeps a stale prior
session's environment.  The result: =xdg-desktop-portal= can't satisfy
its =Requisite=graphical-session.target= and stays dead, so flatpak and
other graphical apps get no =org.freedesktop.portal.Settings= and fall
back to a light theme / no file portal / no screencast.

gowl bridges this from =gowl_compositor_start= (=src/util/gowl-systemd.c=):
on startup it runs =systemctl --user import-environment= for the live
=compositor env (=WAYLAND_DISPLAY=, =XDG_CURRENT_DESKTOP=, ...) and starts
the shipped =gowl-session.target= (which =Wants==graphical-session.target=);
on quit it stops the target.  Everything is best-effort -- a host
without =systemctl= is a silent no-op, and =GOWL_DISABLE_SYSTEMD=1= opts
out entirely.

=make install= places =gowl-session.target= in =$PREFIX/lib/systemd/user=,
which the user manager scans for both =$PREFIX=/usr= and =$PREFIX=/usr/local=.
After installing, =systemctl --user daemon-reload= picks it up; the unit is
started automatically the next time gowl starts.

Use =DESTDIR= for staged installs (packaging):

#+begin_src sh
make install DESTDIR=/tmp/gowl-pkg
#+end_src

** Development Workflow
:PROPERTIES:
:CUSTOM_ID: development-workflow
:END:
A typical development cycle:

#+begin_src sh
# Build debug with all modules
make DEBUG=1

# Run tests
make test

# Run the compositor (in a nested Wayland session or TTY)
./build/debug/gowl

# Run with debug logging
./build/debug/gowl --debug

# Run with a custom config
./build/debug/gowl --config ./my-config.yaml
#+end_src
